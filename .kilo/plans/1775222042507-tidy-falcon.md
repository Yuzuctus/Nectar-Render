# 📋 Plan d'amélioration MD-TO-PDF

## 🎯 Objectif
Finaliser l'audit et proposer un plan d'action complet pour améliorer le projet.

## ✅ Audit terminé - Résultats
- **42 problèmes identifiés** au total
- 7 critiques, 14 importants, 21 mineurs
- Le projet est un excellent MVP avec de très bonnes fondations
- La dette technique est concentrée presque exclusivement sur `app.py`

## 🚀 Pistes d'amélioration proposées

### Court terme (< 1 semaine)
1. ✅ Corriger les 7 problèmes critiques
2. ✅ Corriger les 14 problèmes importants
3. Ajouter un système d'annulation de conversion
4. Ajouter une barre de progression
5. Implémenter le cache de conversion

### Moyen terme (1-4 semaines)
1. Refactoriser `app.py` selon l'architecture proposée
2. Ajouter le mode batch
3. Ajouter le mode watcher
4. Permettre des templates CSS personnalisés

### Long terme (> 1 mois)
1. Mode serveur / API REST
2. Système de plugins
3. Export EPUB
4. Internationalisation

### Petits trucs à fort impact
- Raccourci `Ctrl+S` pour conversion immédiate
- Mémoriser le dernier fichier ouvert
- Bouton "Ouvrir dossier de sortie"
- Afficher la taille PDF dans l'UI

## 📊 Priorisation ROI

| Effort | Impact | Actions |
|---|---|---|
| 🟢 Faible | 🔴 Énorme | Corriger les 7 problèmes critiques |
| 🟡 Moyen | 🟠 Grand | Corriger les 14 problèmes importants |
| 🔴 Élevé | 🟠 Grand | Refactoriser l'architecture |
| 🟢 Faible | 🟡 Moyen | Nettoyage et petites améliorations UI |

## ⚠️ Risques à anticiper
1. Performance sur fichiers > 5000 lignes
2. Memory leak sur sessions longues
3. Compatibilité WeasyPrint entre versions
4. Manque d'internationalisation

## ✅ Conclusion
Ce projet a un énorme potentiel. Pour 2 jours de vibe coding c'est un résultat exceptionnel. La seule chose qui bloque la scalabilité est l'architecture UI qui a atteint ses limites.
